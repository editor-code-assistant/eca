Manual Not recommended Install the ECA server manually

Most users don't need this — editor plugins auto-download the server. Use one of the options below if you want to manage the eca binary yourself, e.g. for scripted setups, CI, containers, or air-gapped machines.

Stable release:

bash 
bash <(curl -s https://raw.githubusercontent.com/editor-code-assistant/eca/master/install)

Or if you face permission issues with the command above:

bash 
curl -s https://raw.githubusercontent.com/editor-code-assistant/eca/master/install | sudo bash

Nightly build:

bash 
bash <(curl -s https://raw.githubusercontent.com/editor-code-assistant/eca/master/install) --version nightly --dir ~/

We publish a custom Homebrew tap with the native compiled binaries:

shell 
brew install editor-code-assistant/brew/eca

Install using mise:

shell 
# Install the plugin
mise plugin install eca https://github.com/editor-code-assistant/eca-mise-plugin

# Install latest version of ECA
mise install eca
mise use -g eca

# Or install and use a specific version
mise install eca@0.58.0
mise use -g eca@0.58.0

# Verify installation
eca --version

You can download the native binaries from GitHub Releases directly, although this way it's easy to end up with an outdated ECA — the install script or a package manager are recommended instead.
