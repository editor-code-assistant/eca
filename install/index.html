    ECA is written in Clojure and compiled into a native binary via GraalVM.

    ℹ️
    Auto install — ECA is already downloaded and updated automatically by all official editor plugins, so you don't need to handle it manually. The tabs below are only for users who want to install ECA themselves.

    Where do you want to use ECA?

    ECA in Emacs

    The eca-emacs plugin bundles and keeps the ECA binary up-to-date automatically. Install the plugin for Emacs and you're done — no separate server install is needed.

    Install eca-emacs plugin Auto-downloads the ECA server on first launch

    ECA in VS Code

    The eca-vscode plugin bundles and keeps the ECA binary up-to-date automatically. Install the plugin for VS Code and you're done — no separate server install is needed.

    Install eca-vscode plugin Auto-downloads the ECA server on first launch

    ECA in Neovim

    The eca-nvim plugin bundles and keeps the ECA binary up-to-date automatically. Install the plugin for Neovim and you're done — no separate server install is needed.

    Install eca-nvim plugin Auto-downloads the ECA server on first launch

    ECA in IntelliJ

    The eca-intellij plugin bundles and keeps the ECA binary up-to-date automatically. Install the plugin for IntelliJ and you're done — no separate server install is needed.

    Install eca-intellij plugin Auto-downloads the ECA server on first launch

    ECA Desktop

    A standalone desktop app for ECA, built with Electron. Download the latest release for your platform directly from GitHub Releases — the ECA server is bundled, and supported formats auto-update in place.

    Choose your architecture

    AppImage Recommended .deb
    AppImage Recommended .deb

    AppImage is the recommended format — it auto-updates in place via electron-updater. .deb does not auto-update; upgrade by downloading a fresh .deb when a new release ships.

    Choose your Mac

    .dmg Recommended .zip
    .dmg Recommended .zip

    Both .dmg and .zip auto-update in place via electron-updater. If macOS Gatekeeper blocks the app on first launch (unsigned builds), right-click the app in Applications and choose Open.

    🔮

    Windows builds are planned

    Native Windows installers aren't shipping yet. In the meantime, you can run ECA through the VS Code or IntelliJ plugin on Windows, or build from source.

    Manual Not recommended Install the ECA server manually

    Most users don't need this — editor plugins auto-download the server. Use one of the options below if you want to manage the eca binary yourself, e.g. for scripted setups, CI, containers, or air-gapped machines.

    Stable release:

    bash 
    bash <(curl -s https://raw.githubusercontent.com/editor-code-assistant/eca/master/install)

    Or if you face permission issues with the command above:

    bash 
    curl -s https://raw.githubusercontent.com/editor-code-assistant/eca/master/install | sudo bash

    Nightly build:

    bash 
    bash <(curl -s https://raw.githubusercontent.com/editor-code-assistant/eca/master/install) --version nightly --dir ~/

    We publish a custom Homebrew tap with the native compiled binaries:

    shell 
    brew install editor-code-assistant/brew/eca

    Install using mise:

    shell 
    # Install the plugin
mise plugin install eca https://github.com/editor-code-assistant/eca-mise-plugin

# Install latest version of ECA
mise install eca
mise use -g eca

# Or install and use a specific version
mise install eca@0.58.0
mise use -g eca@0.58.0

# Verify installation
eca --version

    You can download the native binaries from GitHub Releases directly, although this way it's easy to end up with an outdated ECA — the install script or a package manager are recommended instead.
